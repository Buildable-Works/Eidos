# Audience

Who the product is for, and how each kind of person interacts with it differently. One per product, at the product root as `Audience.md`. This doc and Criteria are the two that decide scope: a spec that serves no one here, or serves a type that does not exist here, is a spec to question.

Keep it to simple blocks of prose. No tables, no personas with stock photos. Just plain description. Suggestive, not prescriptive. See Core Overview on portability.

> Delete this guidance block and the italic prompts as you fill the doc in.

## Audience

_Who uses this product, in a paragraph or two. The whole population it is built for, described plainly. Not segmented yet, just the answer to "who is this for."_

## User types

_How each kind of user interacts differently. One short block of prose per type. The point is the difference: what this type comes for, how they reach for the product, what they ignore. A type earns its place here only when it interacts in a way the others do not._

### Type name

_What this person is here to do and how they go about it._

### Type name

_The next type, described by how it differs from the one above._
